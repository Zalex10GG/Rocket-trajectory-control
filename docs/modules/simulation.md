# Module: `src/simulation.py`

## Overview

`src.simulation` runs the RocketPy closed-loop flight and exports nominal simulation results.

## `simulate_controlled_flight(...)`

This function:

1. Clears existing RocketPy controllers from the rocket object.
2. Finds the controlled `GenericSurface`.
3. Builds a RocketPy private `_Controller` that calls `src.controllers.fin_controller`.
4. Registers the controller with `rocket._add_controllers(...)`.
5. Runs RocketPy `Flight`.
6. Forces evaluation of `flight.apogee_time`.
7. Extracts `flight.solution` into a list of history dictionaries.

`config.terminate_on_apogee` is passed directly to RocketPy `Flight`. The nominal config leaves it false. The gain sweep sets it true.

## Flight History Records

Each `flight_history` sample contains:

- `time_s`
- `position_enu_m`: local ENU position with launch pad as origin
- `position_asl_m`: absolute RocketPy position
- `velocity_enu_m_s`
- `attitude_quaternion`
- `body_rates_rad_s`
- `deltas`
- `q_ref`
- `q_dynamic`
- `mach`

Fin deflections and desired attitude quaternions are reconstructed from controller histories using the latest command at or before the RocketPy solution timestamp.

## `export_results(...)`

If `config.save_results` is true, this function writes to `results/<run_id>/`:

- `metrics.json`
- `flight_summary.csv`
- `flight_history.csv`
- `controller_diagnostics.csv`
- `effective_config.json`
- `rocket_definition.toml`
- `rocket_artifacts.json`
- `simulation/`
- `control/`

The `simulation/` and `control/` directories are generated by `src.plots.generate_all_plots()`. Rocket and motor plots are generated by `src.plots.generate_rocket_creation_plots()`.

## Private API Dependency

The module imports `rocketpy.control.controller._Controller` and calls `rocket._add_controllers(...)`. This is the project-supported integration path and depends on the pinned RocketPy version.
